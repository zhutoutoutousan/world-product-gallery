Thank you for your interest in contributing to the World Product Gallery!

Vielen Dank für Ihr Interesse an der Mitwirkung an der World Product Gallery!

感谢您对世界产品画廊的兴趣！

## How to Contribute / Wie man beiträgt / 如何贡献

### Adding a New Product / Ein neues Produkt hinzufügen / 添加新产品

1. **Choose a product** that is interesting and has a clear supply chain
2. **Research thoroughly** - gather information about:
   - Product specifications
   - Manufacturer details
   - Supply chain (raw materials → production → distribution)
   - Market analysis
   - Geographic locations
3. **Create documentation** following the template structure
4. **Ensure trilingual content** (German, English, Chinese)
5. **Include visualizations** (maps, diagrams, charts)

### Improving Existing Content / Bestehenden Inhalt verbessern / 改进现有内容

- Fix errors or inaccuracies
- Add missing information
- Improve translations
- Enhance visualizations
- Update outdated data

### Code Contributions / Code-Beiträge / 代码贡献

- Improve map generation scripts
- Enhance LaTeX templates
- Add new visualization types
- Optimize build processes

---

## Guidelines / Richtlinien / 指南

### Content Standards / Inhaltsstandards / 内容标准

1. **Accuracy**: All information must be accurate and verifiable
2. **Completeness**: Include all required sections
3. **Trilingual**: Maintain German, English, and Chinese versions
4. **Citations**: Cite sources where appropriate
5. **Visuals**: Include high-quality images and diagrams

### Code Standards / Codestandards / 代码标准

1. **Documentation**: Comment code clearly
2. **Style**: Follow existing code style
3. **Testing**: Test scripts before submitting
4. **Dependencies**: Minimize external dependencies

### Commit Messages / Commit-Nachrichten / 提交消息

Use clear, descriptive commit messages:

```
Add: New product documentation for [Product Name]
Fix: Corrected supply chain data in [Product]
Update: Updated market analysis for [Product]
Improve: Enhanced map visualization for [Product]
```

---

## Submission Process / Einreichungsprozess / 提交流程

1. **Fork** the repository
2. **Create** a feature branch
3. **Make** your changes
4. **Test** thoroughly
5. **Commit** with clear messages
6. **Push** to your fork
7. **Open** a Pull Request

### Pull Request Template / Pull-Request-Vorlage / Pull Request 模板

When opening a PR, please include:

- **Description** of changes
- **Product name** (if adding new product)
- **Screenshots** (if visual changes)
- **Testing** performed
- **Checklist** of completed items
